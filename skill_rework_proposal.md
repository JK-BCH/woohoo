# 스킬 시스템 정리 제안서 (v2.9 시점 · 미구현 — 설계 검토용)

> 문제의식(사용자 피드백): **"스킬이 너무 많아서 쓰는 것만 쓰게 된다."**
> 현재 16종. 전투 메뉴가 길고, 후반에는 하위 호환 스킬이 죽은 슬롯이 된다.
> 이 문서는 **즉각 패치용이 아니라** 다음 밸런스 사이클에서 고를 방향을 정리한 것.

## 1. 현황 진단 — 누가 죽어 있는가

시뮬 AI의 실전 로테이션(§8 방법론)과 스킬 역할을 교차하면 사용률이 뚜렷이 갈린다.

| 격물치지 (Lv2) | 약점 부여 + 딜. 전 구간 1순위 |
| 부동심 (Lv3) | 방어×2 유지. 보스전 생존의 축 |
| 실사구시 (Lv12) | 약점+가드봉인, MP0 |
| 일이관지 (Lv13) | MP½ 한방. 실사→일이관 콤보 |
| 치국평천하 (Lv8) | 다수전 개막 debAtk |
| 수신제가 (Lv21) | 디버프 4축 해제. 3장 메타의 해답 |

## 2. 방향 비교 — 축소 vs 개선

### ❌ 안 B. 물리적 축소(스킬 삭제)는 비추천
- `P.skills` 배열에 구 id가 저장됨 → SKILLS에서 키 삭제 시 **구세이브의 스킬 목록이 깨진다** (세이브 호환 제1원칙 위반).
- applySnapshot에 id 필터를 넣으면 호환 자체는 가능하지만, "쓰던 스킬이 사라졌다"는 경험은 제품 약속 위반에 가깝다.

### ✅ 안 A. 「계열 진화(숨김)」 — 추천, 구현 저렴
상위 스킬을 배우면 하위 호환 스킬이 **전투 메뉴에서 접힌다**. 데이터는 전부 유지.
- `SKILLS`에 `sup:'상위id'` 필드만 추가 (포맷 컨벤션 유지: id/name/mp/lv 순서 뒤에 덧붙임 → verify [7] 무영향).
- 전투 스킬 메뉴 빌드 시 `P.skills.includes(sk.sup)`면 기본 숨김. 메뉴 맨 아래 "…모든 스킬 보기" 토글 1개로 전체 접근 보존 (MP 부족 시 하위 스킬을 쓰는 플레이는 살아 있음).
- 세이브 스냅샷 무변경. 신규 S 플래그 불필요(토글은 전역 변수면 충분).

**진화 라인 제안** (이것만으로 전투 메뉴 16 → 9~10개):
```
지행합일 ──→ 박학심문 ──→ 활연관통     (단일 화력 라인)
해체신서 ──→ 활연관통                  (방어무시 라인)
동도서기 ──→ 법고창신                  (전체기 라인)
온고지신 ──→ 수신제가                  (MP 수급 라인)
경세치용 ──→ (장비/술 버프로 흡수 — 숨김 없이 아래 리워크로 구제도 가능)
```

### ✅ 안 C. 죽은 스킬 정체성 리워크 — A와 병행 (선택)
진화 라인에 안 들어가는 3종은 "다른 일"을 시켜서 살린다.
| 스킬 | 리워크안 | 의도 |
|---|---|---|
| 사단지심 | 4연타 각 타격이 **ATK↓/DEF↓/SPD↓/명중↓ 중 하나를 1턴 부여** (측은·수오·사양·시비=4단 디버프) | 유일한 "디버프 뿌리기" 정체성 |
| 술이부작 | MP 5→3, 반격 배율 1.5→2.0, **반격이 약점 노출 1턴 동반** | 카운터 빌드 한 자리 |
| 경세치용 | 술버프와 **중첩 가능**(곱연산 아닌 합연산 +25%p) + 지속 4턴 | 보스 개막 버프로 차별화 |
- 주의: 리워크는 §8 기준선(보스 승률표)을 흔들 수 있으므로 **건드린 스킬을 로테이션에 넣은 시뮬 재실행 필수**.

### ✅ 안 D. UI만으로 해결 — 최소 비용 차선책
구조를 못 바꿀 때의 보험: 전투 스킬 메뉴를 **최근 사용 3종 상단 고정**(전역 변수, 세이브 불필요) + 나머지 lv순. 구현 30분, 효과는 중간.

## 3. 권장 패키지

1. **1차 (구현비 최소·무위험)**: 안 A 진화 숨김 + 토글. 시뮬 재실행 불필요(수치 무변경).
2. **2차 (다음 밸런스 사이클)**: 안 C 리워크 3종 + 보스 시뮬 재검증.
3. 안 D는 1차에 곁들여도 좋음(숨김 후에도 9개는 남으므로).
4. 신규 스킬 추가는 당분간 동결 — Tenure DLC에서 "진화 라인의 4단계"로만 확장 (예: 활연관통→교학상장).
